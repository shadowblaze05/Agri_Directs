AGRI-DIRECT SYSTEM DOCUMENTATION
================================

OVERVIEW
--------
Agri-Direct is a comprehensive web-based agricultural supply chain management platform that connects farmers, buyers, and administrators in a secure, real-time marketplace. The system enables farmers to upload harvest data, buyers to browse inventory, and administrators to manage the entire platform. Features include location-based mapping, role-based access control, real-time chat, data visualization, and RESTful API integration.

SYSTEM ARCHITECTURE
-------------------
- Backend: Flask (Python web framework)
- Database: SQLite3 with automatic migrations
- Frontend: HTML5, CSS3, Bootstrap 5, JavaScript, jQuery, Select2, Leaflet Maps
- Authentication: Session-based with JWT API support
- Real-time Updates: AJAX polling and dynamic content loading
- Location Services: Philippine Standard Geographic Code (PSGC) API integration
- Mapping: Leaflet.js with OpenStreetMap and Google Maps geocoding fallback

---------------
Tables:
- users: id, username, password (hashed), role, location
- inventory: id, crop_name, quantity, farmer, date_received, location
- messages: id, sender, recipient, message, timestamp

DOWNLOADS, INSTALLATION & SETUP
--------------------------------
Required downloads:
- Python 3.11 or newer
  - Download: https://www.python.org/downloads/
- Git (optional, for cloning repository)
  - Download: https://git-scm.com/downloads
- Code editor (recommended): Visual Studio Code
  - Download: https://code.visualstudio.com/
- Browser: Chrome, Edge, or Firefox

Required Python packages are listed in requirements.txt and installed with pip.

Setup steps:
1. Clone or download the project files to a local folder.
2. Open a terminal in the project folder.
3. Create a Python virtual environment:
   - Windows:
     > python -m venv .venv
   - Linux/Mac:
     > python3 -m venv .venv
4. Activate the virtual environment:
   - Windows:
     > .venv\Scripts\activate
   - Linux/Mac:
     > source .venv/bin/activate
5. Install project dependencies:
   > pip install -r requirements.txt
6. Run the application:
   > python app.py
7. Open the app in a browser at:
   http://127.0.0.1:5000

Optional setup:
- If using location features and geocoding, obtain a Google Maps Geocoding API key.
- Set environment variables for secure keys if deploying in production:
  - SECRET_KEY
  - JWT_SECRET
  - GOOGLE_MAPS_API_KEY

TROUBLESHOOTING
---------------
Common Issues:
- Import errors: Ensure virtual environment is activated and dependencies installed
- Database errors: Delete database.db and restart (will auto-recreate with migrations)
- Upload failures: Check CSV format (crop_name, quantity columns required)
- Location selector not loading: Check internet connection for PSGC API
- Map not displaying: Verify Leaflet.js loading and coordinates
- Chat not working: Check JavaScript console for AJAX errors
- Permission denied: Verify user role and session status
